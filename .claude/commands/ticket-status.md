### ステップ1: 各カテゴリのPROGRESS.mdを読み込み

以下のファイルを読み込んでください：

1. **バグ進捗**:
   - `docs/ticket/bug/PROGRESS.md`

2. **機能追加進捗**:

3. **改善進捗**:

4. **初期構築進捗**（参考）:

### 推奨作業順序

優先度に基づく推奨作業順序：
1. Critical/Highバグ → 即座に対応
2. High機能追加 → ビジネス上重要
3. High改善 → パフォーマンス/セキュリティ
4. Medium/Lowバグ → 余裕があれば対応
5. Medium機能追加/改善 → 計画的に対応
